## Проблема с OAuth авторизацией

К сожалению, стандартный OAuth flow для Avito API не работает с предоставленными Client ID и Client Secret. 
Ошибка: `no Route matched with those values`

## Альтернативное решение

Avito API может требовать другой тип авторизации или дополнительные настройки.

## ⚠️ ВАЖНО: Обновите настройки в Avito!

Перед использованием обязательно обновите Redirect URI в настройках приложения Avito:

### Шаги:

1. Откройте [Портал разработчика Avito](https://developers.avito.ru)
2. Войдите в свой аккаунт
3. Найдите ваше приложение (Client ID: `HHiafcJb3rn8agxhzM8g`)
4. Найдите раздел "Redirect URI" или "URI перенаправления"
5. Добавьте или измените на:
   ```
   http://localhost:5002/callback
   ```
6. Сохраните изменения

---

## Как использовать:

1. ✅ Сервер уже запущен
2. Откройте `http://localhost:5002` в браузере
3. Нажмите "Войти через Avito"
4. Авторизуйтесь в Avito
5. После авторизации вы увидите список чатов
6. Выберите чат и начните отвечать на сообщения

## Если что-то не работает:

### Проблема: "Такой страницы не существует" при авторизации

**Решение:**
1. Убедитесь, что Redirect URI в Avito точно: `http://localhost:5002/callback`
2. Проверьте, что Client ID правильный
3. Откройте `/test-oauth` и проверьте URL авторизации

### Проблема: Сервер не отвечает

**Решение:**
1. Проверьте, что сервер запущен (должен быть процесс python3)
2. Убедитесь, что открываете `http://localhost:5002` (с портом 5002!)
3. Попробуйте `http://127.0.0.1:5002`
